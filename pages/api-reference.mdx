This page contains all available endpoints for the ARP Digital API.

## Overview

The ARP Digital API provides comprehensive functionality for managing remittances, trades, checkouts, and account operations. All endpoints require authentication using API key and signature.

## Authentication

All API requests require authentication using API key and signature. You must include the following headers with each request:

- `X-API-Key`: Your API key
- `X-Timestamp`: Current Unix timestamp in seconds
- `X-Signature`: HMAC SHA256 signature of `apiKey + requestBody + timestamp` using your API secret as the key
- `X-Environment`: `sandbox` or `production`

## Endpoints by Category

### Account Management

- `PUT /profile` - Update user profile
- `POST /change-password` - Change password

### Sender 
- `GET /senders` - Get senders 

## Remittances API

### Recipients

- `GET /gps/api/v1/recipients` - Get recipients
- `POST /gps/api/v1/recipients` - Create recipient
- `GET /gps/api/v1/recipients/{id}` - Get recipient by ID
- `GET /gps/api/v1/recipients/verificationFields` - Get recipient verification fields
- `GET /gps/api/v1/recipients/paymentMethodFields` - Get recipient payment method fields

### Quotes

- `POST /gps/api/v1/quote` - Get exchange rate quote

### Transactions

- `POST /gps/api/v1/transaction` - Execute transaction
- `GET /gps/api/v1/transactions` - Get transactions
- `GET /gps/api/v1/transactions/{id}` - Get transaction by ID
- `GET /gps/api/v1/transactions/report/excel` - Generate transactions report

### Configuration

- `GET /gps/api/v1/countries` - Get countries

## Trade API

### Trades

- `GET otc/api/v1/trades` - Get trades
- `POST otc/api/v1/trades` - Create trades

### Quotes

- `POST otc/api/v1/quotes` - Create exchange rate quote

## Checkout API

- `GET gate/api/v1/health` - Check the health
- `POST gate/api/v1/checkout` - Create checkout
- `GET gate/api/v1/checkout` - Get all checkouts
- `POST gate/api/v1/checkout/broker` - Create broker checkout
- `GET gate/api/v1/checkout/{checkoutId}` - Get checkout by ID
- `GET gate/api/v1/checkout/{checkoutId}/completion-url` - Get the completion URL
- `GET gate/api/v1/checkout/{checkoutId}/receipt` - Get receipt
- `GET gate/api/v1/checkout/buyer/{checkoutId}` - Get buyer
- `GET gate/api/v1/checkout/seller/{checkoutId}` - Get seller
- `POST gate/api/v1/checkout/complete` - Proceed checkout
- `POST gate/api/v1/checkout/penny-test` - Create penny-test
- `POST gate/api/v1/checkout/{checkoutId}/cancel` - Cancel checkout by ID
- `POST gate/api/v1/vaults` - Create vault
- `GET gate/api/v1/vaults/detailed` - Get vault detailed
- `GET gate/api/v1/vaults/basic` - Get basic vault information
